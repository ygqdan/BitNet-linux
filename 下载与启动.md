
### 6.3 如果输出仍出现“乱码/��”：优先检查终端编码

你可以先在运行前设置 UTF-8（通常 Cursor 终端/系统设置本身也是 UTF-8，但可手动兜底）：

```bash
export LANG=zh_CN.UTF-8
export LC_ALL=zh_CN.UTF-8
```

### 6.4 提速版（减少 KV 缓存/上下文）

你前面测试时如果发现“输出很慢”，多半是因为 `llama-cli` 默认/模型元信息导致 KV cache 很大（例如 `n_ctx` 过高）。

可以把上下文和批处理尺寸调小来提速（仍然是中文测试题）：

```bash

cd "/home/srbank/code/BitNet－linux"

export LD_LIBRARY_PATH="$PWD/build/3rdparty/llama.cpp/src:$PWD/build/3rdparty/llama.cpp/ggml/src:$LD_LIBRARY_PATH"

./build/bin/llama-cli \
  -m "models/Qwen2.5-7B-Instruct/qwen2.5-7b-instruct-q4_k_m.gguf" \
  -p "你能干的最复杂的任务是什么？" \
  -n 90 \
  -c 2048 \
  -t 16 -tb 16 \
  -b 512 -ub 128 \
  -fa \
  --cache-type-k q8_0 --cache-type-v q8_0 \
  --temp 0.2 --top-k 30 --top-p 0.9 \
  --repeat-penalty 1.1 --repeat-last-n 128 --min-p 0.05 \
  --no-warmup --no-perf
```

### 6.6 Qwen2.5-14B（Q4_K_M，分片下载+合并）

#### 6.6.1 下载 14B Q4_K_M（3 个分片）

```bash
cd "/home/srbank/code/BitNet－linux"
export HF_ENDPOINT=https://hf-mirror.com

mkdir -p "models/Qwen2.5-14B-Instruct"

huggingface-cli download Qwen/Qwen2.5-14B-Instruct-GGUF \
  --include "qwen2.5-14b-instruct-q4_k_m-00001-of-00003.gguf" \
  --include "qwen2.5-14b-instruct-q4_k_m-00002-of-00003.gguf" \
  --include "qwen2.5-14b-instruct-q4_k_m-00003-of-00003.gguf" \
  --local-dir "models/Qwen2.5-14B-Instruct"
```

#### 6.6.2 合并分片

```bash
cd "/home/srbank/code/BitNet－linux"

./build/bin/llama-gguf-split --merge \
  "models/Qwen2.5-14B-Instruct/qwen2.5-14b-instruct-q4_k_m-00001-of-00003.gguf" \
  "models/Qwen2.5-14B-Instruct/qwen2.5-14b-instruct-q4_k_m.gguf"
```

#### 6.6.3 启动推理（推荐：Flash Attention + q8 KV，明显减轻 KV 带宽）

说明：纯 CPU 跑 14B 仍然偏慢；`-fa` 打开 Flash Attention，`--cache-type-k/v q8_0` 把 KV 从 f16 压到约一半（同样 `-c` 下更少内存与带宽）。线程数可按机器改：`-t $(nproc) -tb $(nproc)`。

**内存偏紧（例如约 16GB RAM、已在用 Swap）时：请优先用下面 6.6.4 的稳妥参数**，避免 `-b 512` + `-c 2048` 在长 prompt 时瞬时吃满内存导致剧烈抖动。

```bash
cd "/home/srbank/code/BitNet－linux"

export LD_LIBRARY_PATH="$PWD/build/3rdparty/llama.cpp/src:$PWD/build/3rdparty/llama.cpp/ggml/src:$LD_LIBRARY_PATH"

./build/bin/llama-cli \
  -m "models/Qwen2.5-14B-Instruct/qwen2.5-14b-instruct-q4_k_m.gguf" \
  -p "你是一个中文技术助手。请用简洁中文回答：Linux 的三个核心特点是什么？" \
  -n 120 \
  -c 2048 \
  -t "$(nproc)" -tb "$(nproc)" \
  -b 512 -ub 128 \
  -fa \
  --cache-type-k q8_0 --cache-type-v q8_0 \
  --temp 0.2 --top-k 30 --top-p 0.9 \
  --repeat-penalty 1.1 --repeat-last-n 128 --min-p 0.05 \
  --no-warmup --no-perf
```

#### 6.6.4 稳妥少抖动（约 16GB 内存 / 已占用 Swap 时推荐）

思路：14B Q4 权重约 8.4GiB，再加大上下文和 `batch` 容易把剩余内存和磁盘 Swap 顶满，表现为一卡一卡。**收紧 `-c` 降 KV 与注意力开销；收紧 `-b` / `-ub` 降长 prompt 预填充时的峰值内存**。仍保留 `-fa` 与 `q8_0` KV。

可选（需 root，整系统行为）：推理前临时降低换出倾向，减轻「边推边被换到 Swap」的抖动，结束后可改回：

```bash
sudo sysctl vm.swappiness=10
# 用完后：sudo sysctl vm.swappiness=60
```

稳妥启动示例（与你当前机型 16 线程、`free` 显示约 16Gi 内存的场景对齐）：

```bash
cd "/home/srbank/code/BitNet－linux"

export LD_LIBRARY_PATH="$PWD/build/3rdparty/llama.cpp/src:$PWD/build/3rdparty/llama.cpp/ggml/src:$LD_LIBRARY_PATH"

./build/bin/llama-cli \
  -m "models/Qwen2.5-14B-Instruct/qwen2.5-14b-instruct-q4_k_m.gguf" \
  -p "你是一个中文技术助手。请用简洁中文回答：Linux 的三个核心特点是什么？" \
  -n 120 \
  -c 1024 \
  -t "$(nproc)" -tb "$(nproc)" \
  -b 256 -ub 64 \
  -fa \
  --cache-type-k q8_0 --cache-type-v q8_0 \
  --temp 0.2 --top-k 30 --top-p 0.9 \
  --repeat-penalty 1.1 --repeat-last-n 128 --min-p 0.05 \
  --no-warmup --no-perf
```

若仍偶发卡顿：先把 `-c` 再降到 `768`；或关其它占内存应用后再跑。一般**不建议**在 16GB 上对 14B 再加 `--mlock`（容易与其它进程抢物理页，反增压力）。

#### 6.6.5 进一步提速（短对话 / 内存更宽裕时）

- 在**内存够用**（例如关闭其它大程序、`free` 里可用内存明显高于模型体积）时，可把 `-c` 提到 `2048`、把 `-b` / `-ub` 提到 `512` / `128`，长 prompt 的预填充会更快。
- 若只追求 decode 更快、不需要长上下文，继续用较短 `-c`（如 `768`～`1024`）往往更划算。
- **内存很宽裕**且希望减少权重被换出导致的抖动时，才可试 `--mlock`；16GB 跑 14B 时通常不优先考虑。
- 若有 NVIDIA GPU 且本机构建了 CUDA，`llama-cli` 可加 `-ngl 99`（或更高）把层卸到显存，速度通常远高于纯 CPU。

```bash
cd "/home/srbank/code/BitNet－linux"

export LD_LIBRARY_PATH="$PWD/build/3rdparty/llama.cpp/src:$PWD/build/3rdparty/llama.cpp/ggml/src:$LD_LIBRARY_PATH"

./build/bin/llama-cli \
  -m "models/Qwen2.5-14B-Instruct/qwen2.5-14b-instruct-q4_k_m.gguf" \
  -p "用一句话介绍 Python。" \
  -n 80 \
  -c 1024 \
  -t "$(nproc)" -tb "$(nproc)" \
  -b 512 -ub 128 \
  -fa \
  --cache-type-k q8_0 --cache-type-v q8_0 \
  --temp 0.2 --top-k 30 --top-p 0.9 \
  --repeat-penalty 1.1 --repeat-last-n 128 --min-p 0.05 \
  --no-warmup --no-perf
```
